### 00_METHODOLOGY_OVERVIEW.md
**「何を、なぜするのか」の理論的背景** | 所要時間: 15分

#### 主要セクション
- 🎯 概要
- 🏗️ 抽出の3つの階層構造
  - マクロレベル（アーキテクチャ・構成）
  - ミッドレベル（関数・クラス・メソッド設計）
  - マイクロレベル（行レベルのコード表現）
- 📊 抽出プロセスの全体フロー
- 🔧 具体的な抽出方法の3つのアプローチ
  - アプローチ1: 統計分析ベース
  - アプローチ2: パターンマッチング・分類ベース
  - アプローチ3: AST（抽象構文木）解析ベース
- ✨ 推奨される統合的アプローチ
- 📝 抽出成果物の形式

#### 推奨読者
- プロジェクト全体を理解したい人
- 理論的背景を学びたい人
- 他のプロジェクトへの適用を考えている人

  - 3.2 スペーシング・区切り文字

#### 推奨読者

**「どうやって抽出するか」の実装手順** | 所要時間: 20分

  - 1.2 クラス・インターフェース設計パターンの分析
  - 1.3 依存関係分析
  - 1.4 エラーハンドリング戦略
  
- ステップ2：ミッドレベル抽出（スクリプト・統計分析ベース）

  - 2.2 パラメータ・戻り値型ヒント分析

- ステップ4：抽出結果の統合
  - 4.1 統一スタイルプロファイルの生成
  
- ステップ5：チェックリストと検証

#### 推奨読者
- 実際に抽出ツールを実装する人

### 03_APPLICATION_TO_QUESTIONS.md
**「抽出した情報をどう活用するか」の問題設計・評価方法** | 所要時間: 25分

#### 主要セクション
- Part 1: 問題設計のフレームワーク
  - 1.1 レベル分けの定義（4段階）
    - Level 1: 基礎（ファンデーション）
    - Level 2: 中級（ミッドレベル）
    - Level 3: 上級（マクロレベル）
    - Level 4: エキスパート（統合）
  
  - 1.2 問題タイプの定義（6種類）
    - type_1_fix_formatting
    - type_2_naming_convention
    - type_3_type_hints
    - type_4_refactor_method
    - type_5_design_pattern
    - type_6_error_handling
  
  - 1.3 問題出題テンプレート（3パターン）
    - Template A: マイクロレベル問題
    - Template B: ミッドレベル問題
    - Template C: マクロレベル問題
  
- Part 2: 解答評価のフレームワーク
  - 2.1 スコアリングモデル（100点満点制）
  - 2.2 詳細評価ルール
    - マイクロレベル評価
    - ミッドレベル評価
    - マクロレベル評価
  - 2.3 評価実行スクリプト（PHP）
  
- Part 3: フィードバック生成
  - 3.1 段階的フィードバック構造
  - 3.2 フィードバック生成例

#### 推奨読者
- 問題設計を担当する人
- 評価システムを構築する人
- チャットボットの質問機能を実装する人
- ユーザーへのフィードバック生成方法を学びたい人

#### このドキュメントを使う場面
- 段階的な学習レベルを設計したい時
- 問題テンプレートが必要な時
- スコアリング基準を決めたい時
- 評価ルールを明確にしたい時
- フィードバック文を生成したい時

#### コピペ可能なコード例
- スコアリングモデル（YAML）
- 評価ルール定義（YAML）
- PHP評価エンジン（CodeEvaluator クラス）
- フィードバック生成例（実例）

## 🎯 使用シーン別ガイド

### シーン1: 「全体的な流れを理解したい」
```
1. README.md を読む（全体像把握）
2. 00_METHODOLOGY_OVERVIEW.md で理論を学ぶ
3. README.md の「ワークフロー」セクションを再読
```
**所要時間: 30分**

### シーン2: 「実際にGuzzleから抽出を行いたい」
```
1. README.md で「読む順序」を確認
2. 01_MULTILAYER_EXTRACTION_FRAMEWORK.md で「何を見るか」を学ぶ
3. 02_IMPLEMENTATION_GUIDE.md の手順に従って実装
4. 各ステップのテンプレートに記録
```
**所要時間: 2-3時間**

### シーン3: 「問題設計・評価システムを構築したい」
```
1. README.md で全体像を把握
2. 03_APPLICATION_TO_QUESTIONS.md で設計・評価方法を学ぶ
3. レベル分けと問題タイプを決定
4. スコアリングルールを実装
5. 評価エンジンを開発
```
**所要時間: 3-4時間**

### シーン4: 「他のPHPプロジェクト（例: Monolog）に適用したい」
```
1. 00_METHODOLOGY_OVERVIEW.md で方法論を確認
2. 01_MULTILAYER_EXTRACTION_FRAMEWORK.md のチェックリストを確認
3. 02_IMPLEMENTATION_GUIDE.md の5段階プロセスを実行
4. 03_APPLICATION_TO_QUESTIONS.md で問題設計
```
**所要時間: 4-6時間**

### シーン5: 「フィードバック生成を改善したい」
```
1. 03_APPLICATION_TO_QUESTIONS.md の Part 3 (フィードバック生成) を詳読
2. フィードバック生成例を参考に実装
3. 段階的フィードバック構造に従う
```
**所要時間: 1-2時間**
