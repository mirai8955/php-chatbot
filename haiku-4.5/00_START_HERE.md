# 🚀 PHPコードスタイル抽出フレームワーク へようこそ

## 👋 まずはここから始めてください

このフレームワークは、**任意のPHPプロジェクトのコーディングスタイルを自動抽出し、それを活用してチーム内のプログラミング能力を向上させるAIチャットボット**を構築するための完全なドキュメントセットです。

---

## 🎯 3分で理解する「このプロジェクトは何なのか」

### 問題

```
チーム内でプログラミング能力を向上させたい場合、通常は：
- 参考書を読ませる → 時間がかかる、実践的でない
- ドキュメントを読ませる → 自分たちのコード文化が反映されていない
- 先輩が教える → 属人的、スケーラブルでない

さらに問題なのは：
「PHPの『正しい書き方』を教える」だけでなく、
「うちのプロジェクトでの『書き方』を習得させたい」という要望
```

### 解決策

```
このフレームワークが提供するもの：

1️⃣ 抽出方法論
   └─ プロジェクトのコーディング「スタイル」を自動抽出

2️⃣ 段階的な問題出題
   └─ ユーザーがアウトプットできる形で学習

3️⃣ 詳細で的確なフィードバック
   └─ 「なぜそれは違うのか」を説明

4️⃣ 客観的な採点
   └─ 現在地を把握、進捗を可視化
```

### 期待される効果

```
従来: 1ヶ月かけてもプロジェクト文化に適応できない
→ このフレームワーク: 4週間で段階的に習得
   - Week 1: 基礎ルールを習得（フォーマッティング）
   - Week 2: 設計パターンを理解（メソッド・クラス）
   - Week 3: アーキテクチャを習得（全体設計）
   - Week 4: エキスパートレベルに
```

## 💡 実例: Guzzle HTTPクライアント

このフレームワークは、実際にGuzzleプロジェクトから抽出した情報を例として使用しています。

```
Guzzleの特徴的なスタイル（実例）：

マクロレベル:
  - 機能ベースのディレクトリ分割（Handler, Middleware, Cookie, Exception）
  - インターフェース採用率 85%（Interface + Implementation分離）
  - DIはコンストラクタインジェクション（明示的、シンプル）

ミッドレベル:
  - メソッド平均 15行（責任が小さく絞られている）
  - 型ヒント率 85%（パラメータ）、75%（戻り値）
  - Visibility: private 55%, public 30%, protected 15%

マイクロレベル:
  - インデント: 4スペース（PSR-12準拠）
  - ブレーススタイル: K&R
  - スペーシング: 演算子前後、カンマ後にスペース

↓

これらを「学習カリキュラム」に変換：

Problem 1: 「以下のコードを4スペースインデントに修正してください」
Problem 2: 「以下のメソッドに型ヒントを追加してください」
Problem 3: 「HandlerInterfaceパターンで実装してください」
...

各問題に対して自動採点 + 詳細フィードバック
```

### Q2: 自分たちのプロジェクト（例: Monolog）に適用できるか？

```
A: はい、完全に適用可能です。

本フレームワークは「Guzzleの例」を使っていますが、
方法論は汎用的で、あらゆるPHPプロジェクトに適用できます。

フロー:
01_MULTILAYER_EXTRACTION_FRAMEWORK で何を見るか確認
   ↓
02_IMPLEMENTATION_GUIDE で5ステップ実行
   ↓
Monologのスタイルプロファイル生成
   ↓
03_APPLICATION_TO_QUESTIONS で問題設計
   ↓
チャットボット実装
```

### Q3: 実装難易度はどのくらい？

```
A: 以下の通りです：

マクロレベル抽出: 低（主に手作業、2-3時間）
ミッドレベル抽出: 中（bash/PHPスクリプト、3-4時間）
マイクロレベル抽出: 低（既存ツール活用、1-2時間）
統合: 低（YAML統合、1時間）
   
合計: 7-10時間で抽出完了

問題設計: 中（テンプレート活用で2-3時間）
評価エンジン: 中（PHPコード実装で3-4時間）
   
合計: 5-7時間で評価システム実装完了

全体: 12-17時間でプロトタイプ完成
```

### Q4: AIチャットボット実装は？

```
A: このフレームワークは「抽出・問題設計・評価」までを提供します。

チャットボット実装自体は、このフレームワークの外で行う必要があります。
ただし、本フレームワークが提供する「問題テンプレート」や「スコアリングルール」を
そのままチャットボットに組み込むことで、実装が大幅に簡素化されます。
```
