## 💡 Monolog スタイルの5大原則（Quick Summary）

| 原則 | 説明 | 例 |
|------|------|-----|
| **1. 厳密な型付与** | すべてのプロパティ・引数・戻り値に型をつける | `protected string $name;` |
| **2. Guard 句** | ネストを最小化し、不正条件で早期 return | `if (!$valid) return;` |
| **3. リソース確実解放** | try-finally で必ずクリーンアップ | `try { } finally { }` |
| **4. インターフェース駆動** | 具象クラスではなくインターフェースに依存 | `HandlerInterface $handler` |
| **5. 詳細なドキュメント** | PHPDoc で型情報・例外・パラメータを記載 | `@param`, `@return`, `@throws` |

**A.** PHP の超有名ログライブラリです。プロダクション品質の高いコーディングスタイルで知られています。

### Q. なぜ Monolog を選んだ？
**A.** 
- 業界標準
- 100+ ファイル、高品質コード
- PHPStan Level 9（最高レベル）
- PSR-12 完全準拠
- チームの学習に最適な規模と複雑度

### Q. うちのチーム固有のスタイルは？

## ⚡ 即座に活用できるチェックリスト

コードレビュー時に以下を確認:

- [ ] `<?php declare(strict_types=1);` が最初の行
- [ ] 全プロパティ・メソッド引数・戻り値に型付与
- [ ] クラス名は PascalCase、メソッド名は camelCase
- [ ] ネスト深度 <= 3（Guard 句で早期 return）
- [ ] try-finally でリソース解放
- [ ] クラス＆public メソッドに PHPDoc
