# コードスタイル抽出に基づくPHPコーチングAI設計

## 概要

このドキュメント群は、任意のPHPプロジェクトのコーディングスタイルを体系的に抽出し、それをベースにユーザーのプログラミング能力を向上させるAIチャットボットを構築するためのアプローチ方法を記述しています。

## 核となる課題

### 1. コードベース理解の壁
- AIがプロジェクト特有のコーディングスタイルを正確に理解する必要がある
- 単なる構文ではなく、プロジェクト固有の「慣習」「パターン」「哲学」を抽出する

### 2. スタイルに基づく能力の定義
- 「このプロジェクトのスタイルに沿ったプログラミング力」とは何か？
- 一般的なPHPスキル + プロジェクト特有のスキル

### 3. 適切な問題の生成
- 抽出したスタイルを反映した実践的な問題を作る
- ユーザーが実務で役立つスキルを身につけられる内容

### 4. 多面的な評価基準
- コーディングガイドラインへの準拠度
- 可読性（プロジェクト基準での）
- 保守性（プロジェクト基準での）
- リソース効率性

## 重要な設計原則

### 1. コンテキスト重視
一般的なベストプラクティスだけでなく、**そのプロジェクトでの文脈**でのベストプラクティスを学習

### 2. 段階的学習
基礎から応用まで、ユーザーのレベルに応じた問題を提供

### 3. 実践的フィードバック
単に正誤を示すだけでなく、「なぜそのスタイルが採用されているか」を説明

### 4. 測定可能性
進捗を定量的に把握できる評価システム

## 使用想定フロー

```
1. プロジェクトコードベースの投入
   ↓
2. 自動スタイル抽出（複数次元で分析）
   ↓
3. スタイルガイド生成
   ↓
4. レベル別問題セット生成
   ↓
5. ユーザーによる問題解答
   ↓
6. 多面的評価と点数化
   ↓
7. 具体的なフィードバック提供
   ↓
8. 次レベルへの推奨と学習パス提示
```

## 成功指標

- **スタイル抽出精度**: プロジェクトメンバーが「これは確かにうちのスタイルだ」と認識できる
- **学習効果**: ユーザーが実際のプロジェクトコードレビューで指摘される頻度が減少
- **エンゲージメント**: ユーザーが継続的に問題に取り組む動機づけができる
- **実務適用**: 学んだことがすぐに実務で使える

## 次のステップ

1. 各ドキュメントを読んで全体像を把握
2. 実際のPHPプロジェクト（例：Guzzle、Monolog）で抽出方法を試験
3. 抽出結果を基に問題プロトタイプを作成
4. 評価システムの妥当性を検証
5. フィードバックループを回して改善
